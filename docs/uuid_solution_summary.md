# 消息ID重复问题 - 解决方案总结

## 问题

用户发送消息后，本地缓存了用户发送的消息，再次查询后出现2条相同消息。

## 根本原因

`sendMessage` 返回的消息ID与 `getSessionMessages` 返回的ID不同，导致合并逻辑无法正确去重。

## 解决方案：客户端生成UUID

### 核心思路

**客户端生成UUID → 发送给远程 → 远程使用客户端UUID → ID保持一致 → 无重复**

### 客户端实现 ✅ (已完成)

**修改文件**：`lib/src/agent/client/agent_proxy.dart`

关键改动：
```dart
Future<String> sendMessage(MessageInput input) async {
  // 1. 客户端生成UUID
  final messageId = input.id ?? const Uuid().v4();
  
  // 2. 创建带ID的input
  final inputWithId = input.copyWith(id: messageId);
  
  // 3. 发送给远程（包含ID）
  final messageData = inputWithId.toMap();
  await _rpcUtil!.sendMessage(SendMessageRequest(
    employeeId: employeeId,
    messageData: messageData,  // 包含id字段
  ));
  
  // 4. 返回客户端生成的ID
  return messageId;
}
```

### 远程服务器要求 ⚠️ (必须配合)

**远程服务器必须使用客户端提供的ID**：

```python
async def agentSendMessage(params):
    message_data = params['messageData']
    
    # ✅ 使用客户端ID
    client_id = message_data.get('id')
    
    # 创建消息
    message = Message(
        id=client_id,  # ← 关键：使用客户端ID
        content=message_data['content'],
        ...
    )
    
    # 持久化
    await database.insert(message)
    
    # 返回相同ID
    return {'messageId': client_id}
```
