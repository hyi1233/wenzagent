# UUID消息ID方案 - 解决消息重复问题

## 问题回顾

用户发送消息后，本地缓存了用户发送的消息，再次查询后出现2条相同消息。原因是：
- `sendMessage` 返回的消息ID与 `getSessionMessages` 返回的ID不同
- 合并逻辑按ID去重，ID不同被认为是两条消息

## 解决方案

### 核心思路：客户端生成UUID

**关键改动**：在 `AgentProxy.sendMessage` 方法中，由客户端生成UUID作为消息ID。

### 实现代码

**修改文件**：`lib/src/agent/client/agent_proxy.dart`

```dart
/// 发送消息
Future<String> sendMessage(MessageInput input) async {
  print('[AgentProxy] sendMessage isLocalMode: $isLocalMode');
  
  // 🔑 关键：在客户端生成UUID，确保远程和本地ID一致
  final messageId = input.id ?? const Uuid().v4();
  print('[AgentProxy] message ID: $messageId (${input.id != null ? "provided by caller" : "generated by client"})');
  
  // 创建带有ID的input副本
  final inputWithId = input.id != null ? input : input.copyWith(id: messageId);
  
  if (isLocalMode && _localAgent != null) {
    print('[AgentProxy] calling local agent sendMessage');
    final returnedId = await _localAgent.sendMessage(inputWithId);
    final pendingMessage = _createPendingMessage(inputWithId, returnedId);
    _pendingMessageQueue.add(pendingMessage);
    return returnedId;
  }
  
  print('[AgentProxy] calling RPC sendMessage');
  // 将 MessageInput 转换为 Map 以便 RPC 传输
  final messageData = inputWithId.toMap();
  final request = SendMessageRequest(
    employeeId: employeeId,
    messageData: messageData,
  );
  final result = await _rpcUtil!.sendMessage(request);
  
  // ✅ 使用客户端生成的ID，而不是远程返回的ID
  // 远程服务器应该使用客户端提供的ID
  final returnedId = result['messageId'] as String? ?? '';
  if (returnedId.isNotEmpty && returnedId != messageId) {
    print('[AgentProxy] ⚠️ Warning: Remote returned different ID ($returnedId) than client generated ($messageId)');
  }
  
  // 将完整消息数据添加到待确认队列
  final pendingMessage = _createPendingMessage(inputWithId, messageId);
  _pendingMessageQueue.add(pendingMessage);
  
  return messageId;
}
```

### 工作原理

```
┌─────────────────────────────────────────────────────────┐
│           客户端发送消息流程                              │
└─────────────────────────────────────────────────────────┘

步骤1: 用户调用 sendMessage(MessageInput)
  ↓
步骤2: AgentProxy 检查 input.id
  ├─ 有值 → 使用提供的ID
  └─ 无值 → 生成 UUID (例如: 4fdd1826-f61d-4bde-969e-a4c57b1e2c0d)
  ↓
步骤3: 创建 input.copyWith(id: messageId)
  ↓
步骤4: 发送给远程服务器 (包含ID)
  messageData: {
    'id': '4fdd1826-f61d-4bde-969e-a4c57b1e2c0d',
    'content': '用户消息',
    'type': 'text',
    ...
  }
  ↓
步骤5: 远程服务器接收消息
  ├─ 使用客户端提供的ID持久化消息
  └─ 返回 {'messageId': '4fdd1826-f61d-4bde-969e-a4c57b1e2c0d'}
  ↓
步骤6: 客户端使用生成的ID缓存消息
  localMessage.id = '4fdd1826-f61d-4bde-969e-a4c57b1e2c0d'
  ↓
步骤7: 调用 getSessionMessages()
  ↓
步骤8: 远程返回消息列表
  [
    {
      'id': '4fdd1826-f61d-4bde-969e-a4c57b1e2c0d',  ← 相同的ID
      'content': '用户消息',
      ...
    }
  ]
  ↓
步骤9: 合并本地和远程消息
  mergedMap[localMsg.id] = localMsg  // ID: 4fdd1826...
  mergedMap[remoteMsg.id] = remoteMsg  // ID相同，覆盖
  ↓
步骤10: 最终结果 → 1条消息 ✅
```

### 关键优势

1. **ID一致性保证** ✅
   - 客户端生成UUID，发送给远程
   - 远程使用客户端提供的ID
   - 本地和远程ID完全一致

2. **防错机制** ✅
   - 如果远程返回不同的ID，客户端会发出警告
   - 客户端仍然使用自己生成的ID
   - 确保本地缓存的ID不变

3. **向后兼容** ✅
   - 支持调用者提供自定义ID
   - 如果不提供，自动生成UUID
   - 不影响现有代码

4. **简单可靠** ✅
   - 不需要复杂的去重逻辑
   - 不依赖时间戳或内容匹配
   - ID唯一且一致

## 远程服务器要求

**重要**：远程服务器必须正确处理客户端提供的ID

### 正确实现（伪代码）
```python
async def agentSendMessage(params):
    message_data = params['messageData']
    
    # ✅ 使用客户端提供的ID
    message_id = message_data.get('id')
    if not message_id:
        # 如果客户端没有提供（不应该发生），生成一个
        message_id = generate_uuid()
    
    # 创建消息实体
    message = Message(
        id=message_id,  # ← 使用客户端的ID
        content=message_data['content'],
        ...
    )
    
    # 持久化消息
    await db.save(message)
    
    # 返回相同的ID
    return {'messageId': message_id}
```

### 错误实现（避免）
```python
async def agentSendMessage(params):
    # ❌ 错误：忽略客户端ID，生成新ID
    message_id = generate_uuid()  # 这会导致ID不一致!
    
    message = Message(
        id=message_id,  # ← 不使用客户端的ID
        ...
    )
    
    await db.save(message)
    
    # 返回不同的ID
    return {'messageId': message_id}  # ← 导致重复消息
```
